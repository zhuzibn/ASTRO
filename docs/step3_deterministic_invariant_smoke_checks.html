1. Summary

Step 3 means adding a small, repeatable MATLAB check that verifies ASTRO's basic output invariants without requiring a full manual inspection.

The goal is not to prove the physics is correct. The goal is to catch structural regressions quickly after future code changes.

Step 2 changed the saved-time contract and updated the accepted current-size baseline to 20 x 30 x 101. Step 3 turns that contract into an automated guard.

2. Checks to include

  • The deterministic fixed atom distribution loads correctly.
  • The output arrays have matching shapes.
  • The first saved time is 0.
  • The last saved time equals runtime.
  • Saved times are strictly increasing.
  • Spin values are finite.
  • Spin normalization remains close to 1.
  • The atom counts match the expected deterministic distribution.
  • The first saved state matches the initialized spin state.

3. Invariant examples

Shape alignment

+numel(t) == size(mmx,3) == size(mmy,3) == size(mmz,3)

Finite spin values

+all(isfinite(mmx(:)))+all(isfinite(mmy(:)))+all(isfinite(mmz(:)))

Spin normalization

+sqrt(mmx.^2 + mmy.^2 + mmz.^2) ≈ 1

4. Why it matters now

The new saved-time contract should not regress. Step 3 prevents future edits from accidentally reintroducing missing t=0, missing runtime, mismatched t/mmx lengths, or duplicated chunk-boundary states.

5. Likely implementation

One likely implementation is a dedicated smoke-test entry point:

+matlab -batch "run('tests/run_smoke_tests.m')"

Or, if keeping the check under benchmark/:

+matlab -batch "run('benchmark/run_smoke_tests.m')"

The MATLAB script should use assert(...) so it exits nonzero when an invariant fails.
