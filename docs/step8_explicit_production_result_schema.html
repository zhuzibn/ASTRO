Step 8 - Explicit Production Result Schema

Branch: codex-d1a8-define-result-schema. Report generated for the current uncommitted working tree.

MATLAB production output Schema version 1.0 Non-destructive output path Smoke-tested

Summary

This improvement replaces production complete-workspace saves with an explicit, versioned astro_result schema. The simulation physics and trajectory arrays remain produced by the existing initialization and RK4 integration path; the change is limited to how production results are packaged and where they are saved.

Before main.m always wrote root final.mat with the entire MATLAB workspace, including transient variables.
After main.m writes one top-level astro_result struct containing documented sections for configuration, atom distribution, initial state, trajectory, timestamps, and provenance.
Output path Callers can set astro_output_file before running main.m; default behavior still targets final.mat.
Regression control Smoke tests now assert the schema shape and ensure workspace-style top-level variables are not leaked.

Behavioral Contract

  • Production MAT-files contain exactly one intended top-level variable: astro_result.
  • astro_result.schema_version is 1.0.
  • Required sections are configuration, atom_distribution, initial_state, trajectory, timestamps, and provenance.
  • trajectory.mx, trajectory.my, trajectory.mz, and trajectory.t preserve the existing saved simulation values.
  • astro_output_file allows verification runs without overwriting root final.mat.
